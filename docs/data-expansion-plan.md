# Data Expansion Plan

## 方針

一度に全期間を取り切らず、25ページ前後の小さい単位で進める。

1. 競馬ラボを新しい順に遡る。
2. 各バッチの後に `npm run audit:data` を実行する。
3. 明らかな崩れがなければ次のバッチへ進む。
4. 競馬ラボが2015年10月まで到達したら、競馬ブック旧アーカイブへ切り替える。

## 競馬ラボ

現在の取り込み済み成功範囲は `focus/724` から `focus/450`。`focus/725` と `focus/726` は現時点で本文が取得できていないため、更新時に再試行する。範囲内の既知404は `461, 487, 488, 493, 500, 501`。

推奨バッチ:

```sh
npm run import:keibalab -- --latest 449 --oldest 400
npm run audit:data
```

画像をローカル保存する場合だけ `--download-images` を付ける。まずはURL参照でデータ構造の整合性を優先する。

## 競馬ブック

競馬ラボが2015年10月まで終わった後、`old.keibado.ne.jp/keibabook/YYMMDD/` の号単位で取り込む。

最初は2015年9月付近から始め、HTML体裁が同じ範囲を確認してから広げる。

## 監査観点

- 競馬ラボで比較用の2枚目以降が残っていないこと
- 単独レースページの写真レース名がページタイトルと一致すること
- 複数レースページではセクション別レース名になっていること
- 写真が存在しない馬、馬が存在しない写真がないこと
- ページ記録の写真枚数と実写真枚数が一致すること
